                      Andrei Despinoiu 5 years ago edited

                      After completing this tutorial the skybox is mirrored on the X axis:

                      View Hide

                      Even when looking at the screenshots from the tutorial, compared to the actual images, the scene is mirrored. While it doesn't matter (at all) for nature environments, if you have text (especially text) or logo that's supposed to be oriented in a certain direction... yeah.

                      The solution is to set the vertex shader like this:
                      #version 330 core
                      layout (location = 0) in vec3 aPos;

                      out vec3 TexCoords;

                      uniform mat4 projection;
                      uniform mat4 view;

                      void main()
                      {
                      //TexCoords = aPos;
                      TexCoords = vec3(-aPos.x, aPos.yz); // Flip X so that words make sense again
                      vec4 pos = projection * view * vec4(aPos, 1.0);
                      gl_Position = pos.xyww;
                      }

                      PS: If you want to test this skybox, it's called "Yokohama at night": http://www.humus.name/Textu...

                          I just want to make a quick note on this as I have been using this method for a while and just now found out that this is wrong. I believe you need to flip the Z instead of the X like so:

                          TexCoords = vec3(aPos.xy, -aPos.z);

                          The reason is that cubemaps in OpenGL use the RenderMan specification and a left-handed coordinate system which is detailed on this page https://www.khronos.org/ope...

                          View Hide

                          You can see that we need to flip only the Z to match the coordinate system. The difference this makes is that the skybox now appears to be rotated 180 degrees around the Y axis. So it's not a very noticeable difference, but should be the correct orientation.

                                    Donlod ThomasD 4 years ago

                                    Im not quite sure what you mean by "instead of X", because as of now this tutorials does not invert anything inside the shader.

                                    Its true that you have to invert X or Z axis. Negating Z is okay. But negating X is also okay. Both do have some inconsistencies. If you negate Z and when you look at world -Z which is the default orientation in these tutorials then you will see the "front" face which is mapped to GL_TEXTURE_CUBE_MAP_POSITIVE_Z.
                                    However when you negate X and look at world +X then you will see the "left" face which is mapped to GL_TEXTURE_CUBE_MAP_NEGATIVE_X.
                                    In both cases the skybox is correct. When looking at "front" and you turn left you will look at "left" and so on. The one of the axis will just not correspond to the cubemap texture binding.

                                    Also if you invert any TexCoord axis for rendering your skybox you also have to invert the corresponding axis of the reflection vector when sampling the skybox to apply reflections/refraction otherwise you see as reflection the opposite side.

                                                                                        Jary 5 years ago edited

                                                                                        For those of you having trouble with the exercise, pay attention to the third point.

                                                                                        By default, we don't explicitly assign a location for the cubemap texture in the shader.

                                                                                        We do, however, access 'uniform samplerCube skybox;'

                                                                                        My understanding is that, if you don't explicitly set a value to this uniform, it defaults to 0.

                                                                                        In our code for the non-exercise part of this lesson, we bound the skybox texture to GL_TEXTURE0 (using glActiveTexture(GL_TEXTURE0) before binding the cube map.

                                                                                        This works fine with the simple cube shader, because it doesn't use any other texture, so the default value of "0" for the sampler is fine (GL_TEXTURE0 is the skybox texture.)

                                                                                        In reality, this is likely not fine, and it might be in your best interest to always explicitly set your uniforms in your shaders. (in this case, glUniform1i(glGetUniformLocation(<yourshadername>.ID, "skybox"), 0);

                                                                                        Now, when we load our model, we're already binding the diffuse texture to GL_TEXTURE0, and the specular texture to GL_TEXTURE1. (The code works out this way, though, with additional textures it may not be that way.)

                                                                                        This won't work! Because we've already bound our skybox to GL_TEXTURE0. To get around this, I did the following;

                                                                                        // Initialize some consts for the skybox texture.
                                                                                        const GLenum skyboxTexture = GL_TEXTURE11;
                                                                                        cosnt int skyboxTextureIdx = 11;

                                                                                        unsigned int loadCubemap(vector<std::string> faces)
                                                                                        {
                                                                                        // Activate whatever texture we've set for skybox
                                                                                        glActiveTexture(skyboxTexture);
                                                                                        [...]
                                                                                        glActiveTexture(GL_TEXTURE0); // unset the skybox texture and default our active texture
                                                                                        }

                                                                                        then, whenever you assign uniforms to your shader (after calling use, to set the program!) add;

                                                                                        glUniform1i(glGetUniformLocation(<yourshadername>.ID, "skybox"), skyboxTextureIdx);

                                                                                        This will work for you, assuming you also set the uniform location for "skybox" for the skybox and the cube as well, in the same manner.

                                                                                                                  Just a note on creating your own skyboxes. I have created my own, you need to render a scene so that each side is a 90 degree field of view (90 x 4 sides = 360, plus top and bottom). You can actually use your camera, and take photos in all directions at 90 degrees and combine them to make a realistic map.                                                                                                                           JoeyDeVries Sensei Dmitry Korovin 8 years ago edited

                                                                                                                          Yes, the texture function for cubemaps expects its 3D texture coordinate (the reflection vector) in world-space so I'd say you have two options:

                                                                                                                          1. You calculate the inverse of the view matrix and multiply this with the reflection vector (GLSL has a built-in inverse function, but it's expensive so it may be better to additionally pass the inverse view matrix as a uniform).

                                                                                                                          2. Or, you (also) get the camera and fragment position in world space and calculate the reflection vectors from these world-space equivalents.

                                                                                                                                      glBindTexture(GL_TEXTURE_2D, textureDepthbuffer);

                                                                                                                                                    Hi there! I tried to do the reflections using view space, and initially I thought everything was going alright. However i noticed that the reflection were wrong, they always were reflecting the same side. For example, if the scene has a rock in the back side, no mater what side I am facing the chromed object it always reflect that rock.

                                                                                                                                                    My vertex shader

                                                                                                                                                    #version 330 core
                                                                                                                                                    layout (location = 0) in vec3 aPos;
                                                                                                                                                    layout (location = 1) in vec3 normal;
                                                                                                                                                    layout (location = 2) in vec2 aTexCoord;


                                                                                                                                                    uniform mat4 Model;
                                                                                                                                                    uniform mat4 View;
                                                                                                                                                    uniform mat4 Projection;

                                                                                                                                                    out vec2 TexCoord;
                                                                                                                                                    out vec3 aNormal;
                                                                                                                                                    out vec3 FragPos;

                                                                                                                                                    void main(){
                                                                                                                                                    aNormal = mat3(transpose(inverse(View * Model))) * normal;
                                                                                                                                                    FragPos = vec3(View * Model * vec4(aPos,1.0));
                                                                                                                                                    gl_Position = Projection * vec4(FragPos,1.0);
                                                                                                                                                    TexCoord = aTexCoord;
                                                                                                                                                    }

                                                                                                                                                    And my Fragment shader

                                                                                                                                                    #version 330 core
                                                                                                                                                    out vec4 FragColor;

                                                                                                                                                    in vec3 FragPos;
                                                                                                                                                    in vec3 aNormal;

                                                                                                                                                    uniform samplerCube skybox;

                                                                                                                                                    void main(){
                                                                                                                                                    vec3 I = normalize(FragPos);
                                                                                                                                                    vec3 R = reflect(I,normalize(aNormal));
                                                                                                                                                    FragColor = vec4(texture(skybox,R).rgb,1.0);
                                                                                                                                                    }
